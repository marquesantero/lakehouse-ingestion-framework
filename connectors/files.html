Connector

Files

Use Spark file readers for data already reachable by the runtime filesystem: Parquet, Delta path, JSON, CSV, ORC, text, Avro and XML.

Example

source:
  type: connector
  connector: parquet
  path: /Volumes/main/landing/orders/
  read:
    schema: "order_id STRING, updated_at TIMESTAMP, amount DOUBLE"
    recursiveFileLookup: true
    pathGlobFilter: "*.parquet"

target:
  catalog: main
  schema: bronze
  table: b_orders

File selection

Prefer Spark-native options such as pathGlobFilter. Use source.read.file_regex when real regex over file name or relative path is required.

read:
  file_regex: "^orders_2026_[0-9]{2}\\.json$"
  file_regex_scope: filename
  file_regex_max_listed: 50000

Operational guidance

  • Declare schemas for many small files to avoid repeated inference cost.
  • XML and Avro support depends on runtime libraries.
  • For continuously arriving files on Databricks, prefer Auto Loader.
