Connector

REST API

Use REST for bounded API extraction, controlled pagination and payload capture. For large recurring loads, land files first and process them with Auto Loader.

Records mode

source:
  type: connector
  connector: rest_api
  request:
    url: https://api.example.com/orders
  response:
    records_path: $.data
  pagination:
    type: cursor
    cursor_param: cursor
    next_cursor_path: $.next
  limits:
    max_pages: 100
    timeout_seconds: 60

Raw payload mode

Use raw mode when the API response is a document and the structure belongs in transform.shape.

source:
  type: connector
  connector: rest_api
  request:
    url: https://eonet.gsfc.nasa.gov/api/v3/events
  response:
    mode: raw
    raw_column: raw_response
  limits:
    max_page_bytes: 10485760

transform:
  shape:
    parse_json:
      - column: raw_response
        alias: payload
        schema: "STRUCT<events:ARRAY<STRUCT<id:STRING,title:STRING>>>"

Operational guidance

  • Always set page, byte, timeout and retry limits.
  • Use secrets for tokens and API keys.
  • Use explicit schemas for heterogeneous payloads.

Raw JSON plus transform.shape

Real API examples showed that connectors should not flatten domain payloads. Capture raw JSON when the response is a document, then parse and explode it declaratively.

response:
  mode: raw
  raw_column: raw_response
transform:
  shape:
    parse_json:
      - column: raw_response
        alias: payload
        schema: "STRUCT<events:ARRAY<STRUCT<id:STRING,title:STRING>>>"
    arrays:
      - path: payload.events
        mode: explode_outer
        alias: event

Use limits.max_pages, limits.max_records and limits.max_page_bytes to keep driver-side API extraction predictable.

How this connector fits the contract

Keep extraction concerns in source, structural normalization in transform, validation in quality_rules and target semantics in mode. This separation keeps examples portable and prevents connector-specific workarounds from becoming hidden business logic.
