Connector

HTTP file

Use http_file for explicit public or authenticated files served through HTTP(S), especially when Spark cannot read HTTPS paths directly.

Example

source:
  type: connector
  connector: http_file
  path: https://example.org/data/orders.csv
  format: csv
  options:
    header: true
  read:
    source_complete: true
  limits:
    timeout_seconds: 60
    retry_attempts: 3
    retry_backoff_seconds: 2

Formats

Supported formats are csv, json, jsonl, ndjson and text. Aliases http_csv, http_json and http_text are available.

Operational guidance

  • Use for small and medium files, not bulk API extraction.
  • The driver downloads the file before Spark materialization; protect it with byte/time limits.
  • For recurring high-volume files, land them in storage and use Auto Loader.
