Connector

JDBC

Use JDBC for relational databases when the runtime has network reachability and the correct driver installed.

Basic auth

source:
  type: connector
  connector: postgres
  options:
    url: "{{ secret:crm/postgres_url }}"
    dbtable: public.orders
    driver: org.postgresql.Driver
  auth:
    type: basic
    username: "{{ secret:crm/user }}"
    password: "{{ secret:crm/password }}"
  incremental:
    watermark_column: updated_at
  read:
    fetchsize: 10000
    partition_column: order_id
    lower_bound: 1
    upper_bound: 1000000
    num_partitions: 8

Amazon RDS/Aurora IAM

auth:
  type: rds_iam
  username: "{{ secret:contractforge-aws/rds_username }}"
  region: us-east-1
  credential_provider: default_chain
  sslmode: require

RDS IAM token generation requires botocore in the Python driver. Install contractforge[aws] or provide botocore in the runtime.

Operational guidance

  • Install the JDBC driver on the cluster or serverless environment.
  • Partition large reads; avoid pulling entire database tables through one connection.
  • Network connectivity is outside the library: VPC routing, firewall, PrivateLink and security groups must be solved by the platform.

Incremental and partitioned reads

Use watermarks for incremental extraction and JDBC partitioning for large tables. Pushdown should narrow the source before Spark reads it.

incremental:
  watermark_column: updated_at
read:
  predicate: "updated_at >= timestamp '2026-01-01'"
  partition_column: id
  lower_bound: 1
  upper_bound: 10000000
  num_partitions: 16
  fetchsize: 10000
Network first

The library can build JDBC options and generate RDS IAM tokens, but routing, firewalls, PrivateLink, VPC peering and security groups are platform prerequisites.

How this connector fits the contract

Keep extraction concerns in source, structural normalization in transform, validation in quality_rules and target semantics in mode. This separation keeps examples portable and prevents connector-specific workarounds from becoming hidden business logic.
