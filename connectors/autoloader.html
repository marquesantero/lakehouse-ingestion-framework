Connector

Auto Loader

Use Databricks Auto Loader when files arrive over time and you need checkpointed, finite ingestion with available_now.

Example

source:
  type: connector
  connector: autoloader
  format: json
  path: /Volumes/main/landing/orders
  read:
    checkpoint_location: /Volumes/main/ops/checkpoints/orders
    schema_location: /Volumes/main/ops/schemas/orders
    available_now: true

target:
  catalog: main
  schema: bronze
  table: b_orders

Operational guidance

  • ContractForge supports finite available-now ingestion, not continuous streaming orchestration.
  • Use stable checkpoint and schema locations outside transient workspace paths.
  • Stream-level metrics are aggregated into ctrl_ingestion_streams.

Available-now execution model

Auto Loader creates an outer stream run and one child ingestion run per micro-batch. The stream table aggregates child rows so dashboards can verify actual files processed.

SELECT stream_run_id, batches_processed, total_rows_read, total_rows_written
FROM main.ops.ctrl_ingestion_streams
ORDER BY start_ts_utc DESC;
Scope

ContractForge targets finite available_now ingestion. Use Databricks streaming jobs directly for long-running continuous stream applications.

How this connector fits the contract

Keep extraction concerns in source, structural normalization in transform, validation in quality_rules and target semantics in mode. This separation keeps examples portable and prevents connector-specific workarounds from becoming hidden business logic.
