Operations

Operate ingestion through evidence, not notebook logs.

Every run should leave enough data to answer what ran, what changed, what failed and how expensive it was operationally.

Recent run health

SELECT
  status,
  target_table,
  mode,
  rows_read,
  rows_written,
  duration_seconds,
  error_message
FROM main.ops.ctrl_ingestion_runs
ORDER BY start_ts_utc DESC
LIMIT 50;

Maintenance commands

contractforge maintenance ctrl-retention --catalog main --ctrl-schema ops --retention-days 30
contractforge maintenance cost-report --catalog main --ctrl-schema ops --dbu-per-hour 2 --currency-per-dbu 0.40

The cost report is operational estimation from run duration and user-provided rates. It is not cloud billing.

Recommended dashboard sections

Health

Success rate and failure trend

Track recent failures by target, mode, connector and runtime type. Use short errors for triage and full traceback tables for diagnosis.

Freshness

SLA and watermark movement

Compare last successful run, expected frequency and watermark progression to detect stalled pipelines.

Quality

Failed and quarantined records

Separate aborting rules from row-level quarantines. The distinction is critical for operational response.

Cost

Rows per second and estimated cost

Use stage durations and row counts to find slow connectors, expensive write modes and low-throughput tables.

Streaming and Auto Loader evidence

Auto Loader available_now records an outer stream run plus child ingestion runs for each micro-batch. Aggregated metrics in ctrl_ingestion_streams should match child run totals.

SELECT
  stream_run_id,
  status,
  batches_processed,
  total_rows_read,
  total_rows_written,
  total_rows_quarantined
FROM main.ops.ctrl_ingestion_streams
ORDER BY start_ts_utc DESC;

Failure triage query

SELECT
  r.run_id,
  r.target_table,
  r.source_connector,
  r.runtime_type,
  r.error_message,
  e.error_type,
  e.stack_trace
FROM main.ops.ctrl_ingestion_runs r
LEFT JOIN main.ops.ctrl_ingestion_errors e
  ON r.run_id = e.run_id
WHERE r.status = 'FAILED'
ORDER BY r.start_ts_utc DESC
LIMIT 20;
